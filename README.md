# ImageRecognition

Курсовой проект по дисциплине **«Алгоритмы и структуры данных»**.  
Система распознаёт изображения цифр и простых геометрических фигур с помощью **kNN** и ускоряет поиск соседей с помощью **KD-Tree**.

## 1. Что реализовано

- Полностью ручная реализация:
  - `KDTree`
  - `kNN`
  - Евклидово расстояние
- Предобработка изображения:
  - загрузка
  - масштабирование до `16x16`
  - grayscale
  - вектор признаков `double[256]`
- Хранение данных в PostgreSQL:
  - признаки (`FLOAT8[]`)
  - метаданные моделей и экспериментов
  - **изображения как бинарные данные (`BYTEA`)**
- WPF-интерфейс с вкладками:
  - подключение к БД
  - работа с моделью
  - справка
- Benchmark:
  - сравнение `KD-Tree` и линейного поиска
  - точность и время выполнения

## 5. Как запустить

### Требования

- .NET SDK 10+
- PostgreSQL 14+
- созданная БД (например `Image_recognition_db`)

### Запуск

```bash
dotnet build ImageRecognition.KDTree.slnx
dotnet run --project ImageRecognition.UI
```

## 6. Как пользоваться приложением

### Шаг 1. Подключение к БД

1. Открой вкладку **«Подключение к БД»**.
2. Заполни `Host / Port / Database / Username / Password`.
3. Нажми **«Подключиться к БД»**.
4. Нажми **«Инициализировать таблицы»**.
5. Нажми **«Проверить записи в БД»**.

### Шаг 2. Добавление обучающих данных

Во вкладке **«Работа с моделью»**:

1. Укажи правильный класс (например `digit_5`).
2. Выбери фото для обучения.
3. Нажми **«Сохранить пример в БД»**.
4. Повтори для нескольких классов и изображений.

Альтернатива: кнопка **«Сгенерировать демо-примеры и загрузить»**.

### Шаг 3. Обучение модели

1. Укажи имя модели.
2. Укажи `k` (например, `3`).
3. Нажми **«Обучить модель»**.

### Шаг 4. Распознавание

1. Выбери фото для определения.
2. Нажми:
   - **«Определить, что на фото (KD-Tree)»** или
   - **«Определить, что на фото (линейно)»**.
3. Смотри блок **«Последний результат»**:
   - `Класс: digit_5` означает, что система распознала цифру 5.

### Шаг 5. Benchmark

Нажми **«Запустить benchmark»** для сравнения:

- точности
- времени KD-Tree
- времени линейного поиска

## 7. Почему система может ошибаться

Главная причина — качество разметки данных.

Если, например, изображение цифры `7` добавить в класс `digit_5`, модель обучится на неверной метке и будет ошибаться.

Рекомендации:

- не смешивать классы;
- добавлять больше разнообразных примеров;
- переобучать модель после добавления новых данных.

## 8. Проверка работоспособности

- расширенная визуализация ошибок и матрица ошибок;
- экспорт отчётов экспериментов.
