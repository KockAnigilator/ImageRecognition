## 3. Введение
Актуальность темы обусловлена необходимостью создавать понятные и объяснимые системы классификации без использования «черных ящиков». Для учебных задач важно показать не только результат, но и внутреннюю работу алгоритма.  
Цель работы: разработать настольное приложение, которое классифицирует изображения цифр и геометрических фигур методом k-ближайших соседей с использованием KD-Tree для ускорения поиска.  
Задачи работы:
- реализовать алгоритмы kNN и KD-Tree вручную;
- разработать конвейер извлечения признаков изображения;
- реализовать хранение данных и результатов в PostgreSQL;
- сравнить производительность KD-Tree и линейного поиска;
- создать интерфейс, удобный для демонстрации и защиты.

## Глава 1. Структуры данных и алгоритмы их обработки

### 1.1 Краткая история и происхождение
Алгоритм kNN относится к методам «обучения по прецедентам». Он не строит явную параметрическую модель, а сравнивает объект с уже известными примерами.  
KD-Tree появилось как структура для эффективного поиска соседей в многомерном пространстве, позволяя отсекать часть кандидатов при поиске.

### 1.2 Описание структуры KD-Tree
KD-Tree строится рекурсивным разбиением пространства по осям. Каждый узел содержит:
- вектор признаков;
- метку класса;
- индекс оси разделения;
- ссылки на левое и правое поддерево.

### 1.3 Описание алгоритма kNN
Для нового объекта вычисляются расстояния до обучающих примеров, затем выбираются `k` ближайших соседей и определяется класс большинством голосов.

### 1.4 Эффективность
- Линейный поиск: `O(n)` на один запрос.
- KD-Tree: в среднем быстрее линейного поиска (особенно при росте `n`), но в худшем случае может деградировать.
- Построение KD-Tree: зависит от способа построения и размерности, в среднем оправдано при многократных запросах классификации.

## Глава 2. Постановка задачи

### 2.1 Требования
- Язык: C#.
- UI: WPF.
- База данных: PostgreSQL.
- Без ML-фреймворков.
- Ручная реализация KD-Tree и kNN.
- Пошаговый интерфейс с обязательным подключением к существующей БД перед работой.

### 2.2 Описание задачи
Необходимо распознавать изображение, приводя его к фиксированному признаковому пространству (`16x16` grayscale), а затем классифицировать по обучающей выборке.

## Глава 3. Выбор средств и их обоснование
- **C#/.NET**: строгая типизация, удобная работа с многослойной архитектурой.
- **WPF**: быстрый desktop UI для учебной демонстрации.
- **PostgreSQL + Npgsql**: надежное хранение данных, в т.ч. `BYTEA` для изображений.
- **MSTest**: встроенное и простое модульное тестирование.

## Глава 4. Проектирование

### 4.1 Архитектура
Проект построен послойно:
- `UI` — взаимодействие с пользователем;
- `Application` — сценарии и оркестрация;
- `Domain` — алгоритмы и сущности;
- `Infrastructure` — SQL и БД.

### 4.2 Схемы
В проект добавлены:

- ER-диаграмма БД;

- диаграмма компонентов;

Интерфейс разделен на вкладки:

Ключевые особенности:

- benchmark и логирование результатов.

## Глава 5. Тестирование
- Модульные тесты проверяют:
  - евклидово расстояние;
  - корректность ближайшего соседа;
  - согласованность линейного и KD-Tree методов;
  - негативные сценарии сервиса (без обучения/без данных).
- В проекте используются три независимых подхода:
  - `MSTest` (основной набор);
  - `xUnit` (математические инварианты расстояния);
  - `NUnit` (smoke-сценарии KD-Tree).
- Ручной чек-лист включает SQL-проверки схемы и записей.

### 5.4 Результаты тестирования производительности
На демонстрационном наборе были получены следующие ориентировочные результаты:
- Accuracy: до `99.54%`;
- Время KD-Tree: `~29.9-41.7 ms`;
- Время линейного поиска: `~66.8-83.4 ms`.

Вывод: при росте выборки режим KD-Tree показывает преимущество по времени поиска.

Типовые ошибки и исправления:
- `created_at does not exist` -> добавлены миграции недостающих колонок;
- ошибочная разметка данных -> введены рекомендации по формированию обучающей выборки;
- отсутствие сохранения изображений -> добавлено хранение в `BYTEA`.

## Выводы
В работе реализована полноценная учебная система распознавания изображений с прозрачной логикой, ручной реализацией алгоритмов и измерением производительности.
